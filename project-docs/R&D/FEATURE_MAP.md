# R&D Feature Map

Status: Candidate

This document captures feature ideas for the wider Gaggi ecosystem.

These are not active roadmap commitments.

Current active work remains:

```text
cleanup + hardening before sync
```

- live settings snapshot

## Shot Analyzer

Status: Working

- telemetry graphs
- profile matching
- analysis tables
- compare mode
- replay/export foundations

## Compare Shots

Status: Present

- pressure comparison

# Easy Expansions From Existing Systems

## Technique Tags

Status: Candidate

Extend existing shot notes with structured fields:

- WDT used
- distribution method
- tamp style
- basket
- puck screen
- filter paper
- grinder model
- grinder setting
- roast date
- open date
- roast level

Value:

- links human puck-prep variables to telemetry outcomes
- low complexity because notes already exist

---

## Bean / Roast Intelligence

Status: Candidate

Build from existing bean notes.

Potential fields:

- roaster
- bean name
- roast date
- open date
- roast level
- process
- origin
- age at shot time

Potential outputs:

- best performance window
- bean ageing curve
- grinder drift notes
- repeatable recipe notes

---

## Cache / Source Indicators

Status: Candidate

Show clearly whether data is:

- live from GaggiMate
- cached locally
- browser/imported
- pending sync
- stale snapshot

Value:

- makes offline-first behaviour obvious
- builds trust in cached data
- reduces user confusion

---

## Profile Matching Improvements

Status: Candidate

Existing profile matching can be made more visible and reliable.

Potential additions:

- show matched profile confidence
- show profile source
- allow manual correction
- remember corrected match locally

---

# Larger Future Features

## Profile Evolution History

Status: Candidate

Track profile versions over time.

Concept:

```text
profile version
→ produced shots
→ produced outcomes
```

Value:

- see which profile version worked best
- compare changes across revisions
- rollback to known-good profile versions later

---

## Extraction Fingerprinting

Status: Candidate

Generate a signature from shot curves.

Inputs:

- pressure curve
- flow curve
- weight curve
- temperature curve
- phase timing
- puck-flow behaviour

Potential uses:

- find similar historical shots
- detect repeated failure patterns
- identify consistency clusters

---

## Telemetry Health Detection

Status: Candidate

Detect possible machine/sensor problems from telemetry.

Examples:

- pressure oscillation
- scale drift
- thermal instability
- flow signal anomalies
- sensor dropouts
- abnormal shot timing

Important:

This should remain observational.

GaggiGo should not become firmware/admin software.

---

## Historical Clustering

Status: Candidate

Group shots by behaviour rather than just metadata.

Potential clusters:

- stable shots
- channeling-prone shots
- slow-flow shots
- fast-flow shots
- high-repeatability shots
- outliers

---

# Ecosystem Features

## GaggiBre Intelligence Viewer

Status: Candidate

Visualise runtime intelligence generated by GaggiBre.

Examples:

- adaptive corrections
- channeling events
- flow interventions
- resistance collapse
- phase behaviour

Requires:

- shared telemetry/event contracts

---

## GaggiStop Telemetry Integration

Status: Candidate

Use GaggiStop as a weight telemetry source feeding the wider dataset.

GaggiGo may show:

- scale connection history
- weight curves
- scale drift
- flow derived from weight
- overshoot behaviour

Important:

GaggiGo must not own stop-control logic.

Machine stop logic belongs inside GaggiMate runtime.

---

## Multi-Device Local Vault

Status: Candidate

Each device running GaggiGo keeps its own offline mirror.

Examples:

- phone cache
- laptop cache
- tablet cache

No account or cloud requirement.

Potential later additions:

- manual export/import
- safe same-network sync
- optional sync queue
